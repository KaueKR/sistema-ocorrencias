4. TEMA E PROPÓSITO
--------------------------------------------------------------------------------

Tema                   : Sistema de Ocorrências
Categoria              : Gestão Institucional / Governança

Propósito do Aplicativo:
  O Sistema de Ocorrências é uma aplicação mobile desenvolvida para digitalizar e
  centralizar o processo de registro, acompanhamento e resolução de ocorrências
  institucionais. O app permite que alunos, professores e gestores registrem
  incidents (incidentes, solicitações ou eventos de atenção) de forma estruturada,
  com rastreabilidade de status e controle de acesso baseado em perfis.

  O sistema elimina fluxos manuais baseados em papel ou e-mail, garantindo
  transparência no ciclo de vida de cada ocorrência — desde a abertura até o
  encerramento — com histórico auditável e diferenciação de responsabilidades por
  nível de acesso (RBAC).

5. TECNOLOGIAS UTILIZADAS
--------------------------------------------------------------------------------

Camada Mobile (Frontend):
  - React Native 0.81.5      — Framework principal para desenvolvimento mobile
  - Expo SDK 54.0.33         — Plataforma de build, device APIs e toolchain
  - React 19.1.0             — Biblioteca de UI e gerenciamento de estado local

Navegação:
  - @react-navigation/native        — Núcleo de navegação
  - @react-navigation/native-stack  — Stack de telas nativas
  - @react-navigation/bottom-tabs   — Navegação por abas inferiores

Backend como Serviço (BaaS):
  - Supabase                 — Plataforma BaaS (PostgreSQL + Auth + Storage)
    · Autenticação de usuários (e-mail/senha, recuperação de senha, MFA-ready)
    · Banco de dados relacional (PostgreSQL) com Row Level Security (RLS)
    · Storage para anexos e mídias de ocorrências

Utilitários e Componentes:
  - @expo/vector-icons (Ionicons) — Biblioteca de ícones
  - react-native-safe-area-context — Tratamento de safe area (notch, barra de status)
  - react-native-screens            — Otimização de performance de telas nativas

Linguagem               : JavaScript (ECMAScript 2022+), JSX
Gerenciador de Pacotes  : npm

   O projeto requer um arquivo `.env` na raiz com as credenciais do Supabase.

   As tabelas do Supabase (perfis, roles, perfil_roles, role_permissoes, permissoes)
   devem estar criadas e com as políticas RLS configuradas antes da execução.
   Scripts de migração: [Inserir caminho para os scripts SQL, se houver]

3. EXECUÇÃO LOCAL:
   Seguir o passo a passo em 03_ORIENTACOES_DE_INSTALACAO_E_EXECUCAO/README_GERAL.md
